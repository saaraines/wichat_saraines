[[section-glossary]]
== Glosario

[options="header",cols="1,3"]
|===
| Término | Definición
| *API* | Application Programming Interface. Interfaz que permite la comunicación entre diferentes servicios o aplicaciones.
| *CORS* | Cross-Origin Resource Sharing. Mecanismo que permite solicitudes HTTP entre diferentes orígenes.
| *E2E* | End-to-End. Tipo de testing que prueba el flujo completo del sistema desde la perspectiva del usuario.
| *Gemini API* | API de Google que proporciona modelos de IA generativa, usado para el sistema de pistas.
| *Hint* | Pista generada por IA que ayuda al usuario a responder preguntas sin revelar la respuesta directa.
| *Jest* | Framework de testing para JavaScript usado para tests unitarios.
| *JWT* | JSON Web Token. Estándar para crear tokens de acceso que permiten autenticación.
| *LLM* | Large Language Model. Modelo de IA usado para generar pistas durante el juego.
| *Material-UI* | Biblioteca de componentes React que implementa Material Design de Google.
| *Microservicio* | Arquitectura donde la aplicación se divide en servicios pequeños e independientes.
| *MongoDB* | Base de datos NoSQL orientada a documentos que almacena los datos del sistema.
| *Puppeteer* | Librería de Node.js que permite controlar navegadores Chrome para testing automatizado.
| *React* | Librería de JavaScript para construir interfaces de usuario interactivas.
| *REST* | Representational State Transfer. Estilo arquitectónico para APIs web usando HTTP.
| *Wikidata* | Base de conocimiento libre de donde se obtienen los datos para las preguntas de trivia.
|===
